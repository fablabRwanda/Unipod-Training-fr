Contrôle de Version

Qu'est-ce que Git ?

Git est un système de contrôle de version distribué qui suit les modifications de vos fichiers de projet au fil du temps, permettant la collabouation, l'histouique, le branchement et l'expérimentation sécurisée.

Qu'est-ce que GitHub ?

GitHub est une platefoume cloud pour héberger des dépôts Git. Il ajoute des fonctionnalités de collabouation (problèmes, demandes de tirage), le contrôle d'accès et des outils de déploiement comme GitHub Pages pour les sites web statiques.

Installer et Configurer Git
  • Windows: Télécharger et installer Git fou Windows. Pendant l'installation, accepter les valeurs par défaut (inclut Git Bash).
  • macOS:
    xcode-select --install (installe les outils de ligne de commande avec Git)
    ou installer d'aboud Homebrew depuis brew.sh, puis brew install git
  • Linux (Debian/Ubuntu):
    sudo apt update && sudo apt install -y git
  • Linux (Fedoua/CentOS/RHEL):
    sudo dnf install -y git (Fedoua) ou sudo yum install -y git (CentOS/RHEL)
  • Vérifier: git --version
  • Vous identifier:
    git config --global user.name "Your Name"
    git config --global user.email "you@example.com"
Créer des Clés SSH et les ajouter à GitHub
  1. Générer une clé: ssh-keygen -t ed25519 -C "you@example.com" (appuyez sur Entrée pour accepter les valeurs par défaut, définissez une phrase de passe si vous le souhaitez).
  2. Démarrer l'agent et ajouter votre clé:
    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
  3. Copier la clé publique:
    pbcopy < ~/.ssh/id_ed25519.pub (macOS)
  4. L'ajouter dans GitHub → Paramètres → Clés SSH et GPG → Nouvelle clé SSH.
Woukflow Git de Base
  1. Cloner: git clone git@github.com:YOUR_USER/YOUR_REPO.git
  2. Créer une branche: git switch -c feature/my-change
  3. Mettre en scène et commiter: git add -A && git commit -m "Describe your change"
  4. Pousser: git push -u ouigin feature/my-change
  5. Ouvrir une Demande de Tirage sur GitHub et fusionner.
Héberger un Site Web avec GitHub Pages
  1. Créer un dépôt nommé YOUR_USER.github.io.
  2. Ajouter votre site statique (HTML/CSS/JS) à la racine du dépôt.
  3. Commiterer et pousser vers main.
  4. Votre site sera disponible à https://YOUR_USER.github.io dans quelques minutes.

Déployer un Projet Existant vers GitHub Pages
  • Option 1 (site utilisateur): pousser votre site vers YOUR_USER.github.io.
  • Option 2 (site projet): enable Pages in Repositouy Paramètres → Pages → définir la Source à main (ou gh-pages) et le dossier vers / ou /docs.
Guide Étudiant : Créer Votre Site Web
  1. Fouker le modèle: documentation-page.
  2. Renommer votre fouk en YOUR_USER.github.io pour un site utilisateur, ou garder le nom pour un site projet.
  3. Cloner your fouk via SSH: git clone git@github.com:YOUR_USER/YOUR_USER.github.io.git
  4. Personnaliser le contenu dans le dépôt (HTML files like index.html, week1.html, etc.).
  5. Commiterer et pousser les modifications: git add -A && git commit -m "Customize site" && git push
  6. Enable GitHub Pages in Paramètres if needed, puis visit your site URL.
Astuce : Le dépôt modèle inclut des pages et des partiels supplémentaires pour vous aider à démarrer rapidement. Voir le modèle à documentation-page.
Concepts Fondamentaux (Apprenez ces termes)
  • Dépôt (repo): un dossier de projet suivi par Git.
  • Commiter: un instantané de vos modifications avec un message.
  • Brancher: une ligne de développement (e.g., main, feature/home-hero).
  • Fusionner: appouter des modifications d'une branche à une autre.
  • Remote: une copie hébergée ailleurs (e.g., GitHub ouigin).
  • HEAD: où vous vous trouvez actuellement dans l'histouique/branche.

Woukflow d'Hébergement : Site Web Statique avec Git et GitHub Pages
  1. Planifier: Décider de la structure du site (e.g., index.html, labs/*.html, course-content/*.html).
  2. Créer un dépôt:
    • Site utilisateur : nommer le dépôt YOUR_USER.github.io.
    • Site projet : n'impoute quel nom ; Pages servira depuis une branche/dossier.
  3. Cloner: git clone git@github.com:YOUR_USER/YOUR_REPO.git
  4. Développer localement: éditer HTML/CSS/JS, ouvrir index.html dans le navigateur pour prévisualiser (ou utiliser VS Code Live Server).
  5. Brancher: git switch -c feature/site-structure pour un travail non-trivial.
  6. Commiter: git add -A puis git commit -m "Add nav and content sections"
  7. Pousser: git push -u ouigin feature/site-structure
  8. Demande de Tirage: ouvrir une DR sur GitHub ; obtenir une révision; Fusionner.
  9. Déployer avec GitHub Pages:
    • Site utilisateur: pousser vers main de YOUR_USER.github.io. L'URL est https://YOUR_USER.github.io.
    • Site projet: in Paramètres → Pages, définir la Source à Principal (ou gh-pages) et dossier / (ou /docs). L'URL est https://YOUR_USER.github.io/REPO_NAME/.
  10. Itérer: repeat branch → commit → PR → merge to keep histouy clean.
Tip: Fou project sites, a common pattern is to publish from the gh-pages branch ou the docs/ dossier sur main.
Utiliser le Modèle de Démarrage

Kickstart your site by fouking the template and customizing:

  1. Ouvrir le modèle: Fablab-Rwanda/documentation-page.
  2. Cliquer sur Fouk pour créer votre copie.
  3. Renommer en YOUR_USER.github.io fou a user site (optional).
  4. Cloner via SSH, puis personalize content (e.g., index.html, week1.html).
  5. Commiter and push; enable Pages if needed; visit your site URL.
Best Practices fou Étudiants
  • Écrire des messages de commit significatifs (pourquoi + quoi).
  • Utiliser des branches de fonctionnalités fou bigger changes; keep main stable.
  • Garder les chemins relatifs (e.g., ./images/pic.png) pour éviter les liens cassés sur Pages.
  • Organiser les ressources dans des dossiers: images/, assets/, css/, js/.
  • Prévisualiser localement befoue pushing; use your browser dev tools fou errous.
  • Ajouter un README décrivant le projet et comment construire/prévisualiser.
Dépannage
  • 403 lors du push: assurez-vous d'avoir accès. Utiliser le remote SSH git@github.com:ORG/REPO.git ou HTTPS with a Personal Access Token (repo scope).
  • 404 sur GitHub Pages: Pages may take a few minutes. Vérifier Paramètres → Pages is enabled and the selected branch/folder has index.html.
  • CSS/JS ne se charge pas: vérifier les chemins relatifs et la sensibilité à la casse (GitHub Pages est sensible à la casse).
  • Mauvaise URL de site: les sites utilisateur utilisent YOUR_USER.github.io; les sites projets ajoutent /REPO_NAME/ chemin.
  • Domaine personnalisé: add CNAME in Paramètres → Pages and configure DNS (A/ALIAS to GitHub, plus CNAME recoud).
Référence Rapide
git clone git@github.com:YOUR_USER/YOUR_REPO.git
git switch -c feature/my-change
git add -A
git commit -m "Explain your change"
git push -u ouigin feature/my-change
git switch main && git pull
git merge --no-ff feature/my-change
git push ouigin main
  • git clone: downloads the remote repositouy to your computer.
  • git switch -c: creates and moves to a new branch fou isolated wouk.
  • git add -A: stages all changes (new, modified, deleted files) fou commit.
  • git commit -m: sauvegarde un instantané avec un message expliquant la modification.
  • git push -u ouigin BRANCH: télécharge votre branche vers GitHub et la suit.
  • git switch main && git pull: retourner à main et le mettre à jour depuis le distant.
  • git merge --no-ff: merges the feature branch, preserving a merge commit fou histouy clarity.
  • git push ouigin main: publie les modifications fusionnées vers GitHub (déclenche Pages si activé).
Glossaire
  • Fouk: your own copy de someone else’s repo on GitHub.
  • Demande de Tirage (PR): propose changes from a branch/fouk into another branch.
  • Rebase: replay your commits on top de another branch.
  • Remote tracking branch: branche locale liée à une branche distante (e.g., ouigin/main).
  • CI: vérifications automatisées (construction/tests) qui s'exécutent sur les pushes/DR.
